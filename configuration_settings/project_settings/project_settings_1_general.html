Easy CHM 工程设置 - 常规对话框

您可以通过选取程序文件菜单下的"编译指定工程"并选取"常规"跳格看到这个对话框。关于这个对话框的使用请看下面的具体描述. 

 

工程设置 /工程的根目录:
显示或修改CHM工程文件的根目录。
注意：
工程的根目录必须存在，否则将无法生成CHM。
工程的根目录通常就是您在“新建工程”时指定的开始目录，如果您“新建工程”后修改了开始目录的名称或者移动开始目录到了一个新的位置，这时在工程的根目录右边会出现一个目录选择按钮，在这里您可以重新指定开始目录的新位置（当然，您也可以在打开工程文件后，通过选取“文件”菜单下的"修改工程文件的根目录为..."来改变当前工程文件的根目录）。


工程设置 / CHM的标题:
CHM的标题会显示在CHM文件的标题栏上 - 如果使用您的CHM的用户有可能使用其他语言的操作系统,建议您使用英文的CHM标题 - 不然在其他语言的操作系统里,您的CHM的标题会被显示为统一的 Htmlhelp。

工程设置 / CHM的第一页:
决定CHM文件被打开后首先显示哪个页面 - 比如一个网页导航页、版权页或者一个欢迎页面。

 工程设置 / CHM的主页:
决定当CHM工具栏上的“主页”按钮被按下时显示哪个页面。

 工程设置 / CHM保存为:
决定生成的CHM的文件名及保存位置。

 工程设置 / CHM设置:
按下CHM设置按钮您将打开CHM设置对话框。这里有丰富的选项供您选择。

工程设置 / 应用:
马上保存当前的工程设置。

工程设置 / 帮助:
打开帮助文件并显示关于“工程设置”的更多介绍。

　

注意：

在工程设置里如果某个选项呈现为红色，则表现指定的文件或目录不存在，需要重新指定，否则将无法进行编译。
同样，如果按钮生成 CHM无法使用，则说明工程设置里的某个文件或目录不存在 - 特别是工程的根目录不存在，必须重新指定。

在工程设置里某个选项呈现为红色，通常是由于您在资源管理器里修改了工程文件根目录的名称、修改了CHM的第一页或CHM的主页所指向的网页的文件名造成的。

　

生成 CHM:
真正开始编译制作.CHM格式的文件。

查看 CHM:
打开已经编译制作好的CHM文件。

取消:
退出当前对话框，返回主界面。
